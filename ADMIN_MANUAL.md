# Інструкція адміністратора АРМА — Реєстр активів

## 1. Як відкрити адмін-панель

Адмін-панель — окремий файл `admin.html` у папці сайту.

**Локально:** відкрити через сервер (не файл://)
```bash
python3 -m http.server 8080
# Потім: http://localhost:8080/admin.html
```

**На хостингу:** `https://ваш-домен.ua/admin.html`

> ⚠️ Рекомендується захистити адмін-панель паролем через `.htaccess` (Basic Auth):
> ```apache
> AuthType Basic
> AuthName "ARMA Admin"
> AuthUserFile /var/www/.htpasswd
> Require valid-user
> ```

---

## 2. Структура бази даних

Всі дані зберігаються у браузері у **localStorage** під ключем `arma_admin_db_v2`. При першому відкритті `admin.html` дані автоматично завантажуються з JSON-файлів сервера і зберігаються локально. Подальші зміни зберігаються тільки в браузері.

**⚠️ Важливо:** щоб зміни з'явились на публічному сайті — потрібно **експортувати JSON** і завантажити його на сервер (детально нижче).

## 4. Редагування запису

1. Знайдіть запис у таблиці
2. Натисніть кнопку **"✏ Редаг."** у рядку
3. Відкриється форма з усіма полями
4. Внесіть зміни
5. Натисніть **"💾 Зберегти"**

**Поля форми:**

| Поле | Опис |
|------|------|
| ID Активу | Унікальний ідентифікатор (не змінювати!) |
| Тип активу | Нерухомість / Земля / Транспорт тощо |
| Вид активу | Детальний вид (напр. "Квартира") |
| Опис | Повний текст опису з реєстру |
| Область/Район/Місто | Адресні поля (будуть автоматично об'єднані) |
| Тип вулиці + Вулиця + Будинок | Точна адреса |
| Кадастровий номер | Формат: `XXXXXXXXXX:XX:XXX:XXXX` |
| Стан арешту | Вибір зі списку |
| Складність | Простий / Складний |
| Власник / Управитель | Назва юридичної або фізичної особи |
| Вартість / Валюта | Числове значення та валюта |
| Широта / Довгота | GPS координати (якщо відомі точно) |

---

## 5. Додавання нового активу

1. Натисніть кнопку **"＋ Додати актив"** у верхньому правому куті
2. Заповніть форму (мінімум: ID та Опис)
3. Натисніть **"💾 Зберегти"**

**Адреса автоматично збирається** з полів Область + Район + Місто + Тип вулиці + Вулиця + Будинок.

**Геокодування:** якщо координати не заповнені вручну — при відображенні на карті сайт автоматично визначить їх через Nominatim.

---

## 6. Видалення запису

1. Знайдіть запис у таблиці
2. Натисніть **"🗑"** у рядку
3. Підтвердіть видалення у вікні
4. Запис буде видалено

## 7. Імпорт з Excel/CSV

### Підготовка файлу

1. Натисніть **"📋 Шаблон XLS"** — завантажиться шаблон з правильними колонками
2. Заповніть дані у шаблоні (або використайте наявний XLSX/CSV)
3. Зверніть увагу на аркуш "Інструкція" в шаблоні

### Завантаження файлу

1. Натисніть **"⬆ Імпорт XLS"**
2. Перетягніть файл у вікно або натисніть для вибору
3. Перегляньте попередній перегляд (перші 5 рядків)
4. Перевірте інформацію: "Нові: X · Оновлення: Y"
5. Натисніть **"⬆ Імпортувати"**

**Поведінка при імпорті:**
- Якщо ID вже існує → запис **оновлюється**
- Якщо ID новий → запис **додається**
- Наявні записи без змін → **залишаються**

### Обов'язкові колонки

| Колонка | Тип | Приклад |
|---------|-----|---------|
| ID Активу | Текст | 350/0102/2020 |
| Тип активу | Вибір | Нерухомість |
| Вид активу | Текст | Квартира |
| Опис | Текст | Квартира за адресою... |
| Стан арешту | Вибір | Арештовано |

## 9. Як оновити дані на публічному сайті

**Проблема:** адмін-панель зберігає зміни в браузері, але публічний сайт читає JSON-файли з сервера.

**Рішення:**

### Варіант А — Через JSON (для розробників)

Після внесення змін в адмін-панелі відправте оновлений JSON на сервер програматично. У майбутньому можна додати кнопку "Опублікувати" (потребує backend PHP).

### Варіант Б — Через Excel (рекомендовано)

1. Внесіть всі зміни в адмін-панелі
2. Експортуйте в Excel (`⬇ Excel`)
3. Відкрийте Excel → відфільтруйте потрібний аркуш
4. Завантажте на Python-сервер для перетворення в JSON

> **Найпростіший варіант:** надайте виправлений CSV/Excel розробнику → він запустить скрипт → перегенерує JSON → завантажить на сервер.

### Варіант В — Пряме редагування JSON

Для невеликих виправлень можна безпосередньо відредагувати JSON-файл на сервері через cPanel → Файловий менеджер.

## 10. Як пов'язаний файл адрес з сайтом

**Файл `ADDRESS_ENRICHED_v2.csv`** — це CSV для перевірки та ручного виправлення адрес. Він НЕ є частиною сайту напряму.

**Зв'язок такий:**
```
ADDRESS_ENRICHED_v2.csv  →  (ручна перевірка/виправлення)
                         →  Python скрипт генерує realestate.json
                         →  realestate.json завантажується на сервер
                         →  Сайт читає realestate.json
```

**Поля в CSV:**
- `P_Повна_адреса` — автоматично розпарсена адреса
- `P_Місто`, `P_Вулиця`, `P_Будинок` — окремі поля
- `P_Geocode_Query` — запит для Nominatim геокодування
- `P_Кадастр` — кадастровий номер з опису
- `P_Метод` — яким способом знайдена адреса:
  - `P1_inverted` — "по вул. X в м. Y"
  - `P2_locative` — "м.Одеса по вул.X"
  - `P4_inline` — "м. Київ вул. X N"
  - `P5_standard` — стандартний "за адресою:"
  - `P6_arc` — АР Крим

**Виправлення адреси:**
1. Відкрийте `ADDRESS_ENRICHED_v2.csv` в Excel
2. Знайдіть ID активу з помилковою адресою
3. Виправте колонки `P_Місто`, `P_Вулиця`, `P_Будинок`, `P_Geocode_Query`
4. Збережіть CSV
5. Надайте файл розробнику для перегенерації JSON

## 11. Кадастрові ділянки

На вкладці **Земля** при відкритті картки ділянки з кадастровим номером:
1. Маркер переміщується на приблизне місце
2. Система намагається завантажити межі ділянки через API `kadastr.live`
3. Контур відображається на карті
4. Є посилання на `kadastrova-karta.com/dilyanka/НОМЕР`

**Якщо межі не завантажились:** перейдіть на `kadastrova-karta.com` та введіть кадастровий номер вручну.

## 12. Резервне копіювання

**Регулярно:** Натискайте "⬇ Excel" та зберігайте файл. Це ваш бекап.
