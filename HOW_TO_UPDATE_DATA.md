# Як оновити дані на сайті — інструкція для адміністратора

## Спочатку: ЯК ВЗАГАЛІ ВЛАШТОВАНІ ДАНІ

Сайт АРМА — це **статичний сайт** на GitHub Pages. У нього **немає бази даних**.
Всі дані лежать у JSON-файлах у папці сайту:

| Файл | Що містить |
|------|-----------|
| `realestate.json` | Всі об'єкти нерухомості (квартири, будівлі) |
| `land.json` | Всі земельні ділянки |
| `transport.json` | Транспорт |
| `corp.json` | Корпоративні права |
| `money.json` | Грошові активи |
| `movable.json` | Рухоме майно |
| `other.json` | Інше майно |
| `stats.json` | Загальна статистика для головної |
| `cases_compact.json` | Судові справи |

## ❗ Чому ваші зміни в JSON не відображались

Дві причини:

**1. Кеш браузера.**
Браузер запам'ятовує JSON-файли і не оновлює їх навіть якщо ви залили нові.

**Рішення:** Після завантаження нових файлів на сервер натисніть **`Ctrl+Shift+R`** (Windows) або **`Cmd+Shift+R`** (Mac) для жорсткого перезавантаження.

**2. Кеш GitHub Pages CDN.**
Якщо хостинг — GitHub Pages, він кешує файли до **10 хвилин** на CDN-серверах. Ваші зміни просто ще не дійшли до відвідувачів.

**Рішення:** Зачекайте 10 хвилин після push в GitHub, потім перевіряйте.

**3. Невідповідність ID.**
Якщо ви змінили запис, але `id` в JSON не співпадає з тим, що ви шукаєте — сайт показує старий запис із кешу.

## ✅ ПРАВИЛЬНИЙ ШЛЯХ ОНОВЛЕННЯ ДАНИХ

### Варіант A: Через адмін-панель (для невеликих змін)

1. Відкрити `admin.html` → ввести пароль `arma2024`
2. Знайти потрібний запис через пошук
3. Натиснути "Редагувати"
4. Зберегти зміни
5. Натиснути **"⬇ Excel"** → завантажиться `arma_db_export.xlsx`
6. Натиснути **"📦 Експорт JSON"** (якщо такої кнопки немає — будемо додавати)
7. Замінити JSON-файли на сервері

⚠ Адмін-панель зберігає зміни в `localStorage` браузера, а не на сервер. Тому після редагування **обов'язково експортуйте** і **залийте** на GitHub.

### Варіант B: Прямий редагувач JSON (для масових змін)

1. Завантажити `realestate.json` (або `land.json`) з GitHub
2. Відкрити в редакторі (VS Code, Sublime, Notepad++)
3. Знайти запис за `id` (Ctrl+F → шукати `"id":"3816/0101/2020"`)
4. Змінити потрібні поля
5. Зберегти файл
6. Завантажити на GitHub (commit + push)
7. Зачекати 5-10 хвилин (CDN кеш) + `Ctrl+Shift+R`

### Варіант C: Через генерацію з Excel (для повного оновлення)

Якщо ви маєте новий `Інвентаризація.xlsx` файл:

1. Передайте файл розробнику
2. Розробник запустить скрипт парсингу
3. Згенерує нові JSON-файли
4. Залиє на GitHub

## 🔑 Структура запису в realestate.json

```json
{
  "id": "3816/0101/2020",
  "addr": "м. Київ, Печерський район, вул. Щорса, буд. 26-А, кв. 3",
  "oblast": "м. Київ",
  "city": "Київ",
  "district": "Печерський район",
  "street": "Щорса",
  "house": "26-А",
  "apt": "3",
  "kadastr": "8000000000:90:001:1234",
  "lat": 50.43521,
  "lng": 30.54123,
  "geo_quality": "geocoded",
  "own": "ТОВ Назва",
  "arr": "Арештовано",
  "zone": "Жовта зона",
  "desc": "Опис активу...",
  "court_cases": ["757/3035/19-к"],
  "primary_verdict": "Передача в управління АРМА"
}
```

**Які поля можна змінювати:**
- `addr`, `oblast`, `city`, `district`, `street`, `house`, `apt` — адреса
- `lat`, `lng` — координати (важливо: повинні відповідати області!)
- `geo_quality` — `"exact"`/`"geocoded"`/`"city"`/`"oblast"`/`"approximate"`
- `own` — власник
- `arr` — стан арешту
- `zone` — зонування
- `desc` — опис

**Які поля НЕ ТРЕБА чіпати:**
- `id` — ключ запису (зламає посилання)
- `court_cases`, `primary_verdict` — генеруються автоматично

## 🚀 Швидкий тест — як перевірити що зміни прийшли

1. Відкрити сайт → знайти ваш об'єкт через пошук
2. Натиснути `F12` → вкладка **Network** → галочка **Disable cache**
3. `Ctrl+Shift+R` → передивіться що завантажується свіжий `realestate.json`
4. Перевірте, що дані змінились

Якщо все одно старі дані — значить файл на сервері не оновився.
